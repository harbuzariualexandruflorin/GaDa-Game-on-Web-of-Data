1. Introduction

GaDa stands for Game on Web of Data. It is a service-oriented game designed for both quiz, and card games and Marvel, Star Trek, Star Wars, and Pokemon lovers alike, where the player is matched up against the computer in a game of cards that requires strategic card management and critical thinking. At the end of each round, the player is asked to take a quiz that will determine the strength of the cards that he will be given. The cards represent various characters ranging from tiny pokémon to well-established superheroes and the goal for the player is to amass more points than the opponent with the cards he is dealt with. Then, both the player and the computer will take turns placing cards that have a higher power level than the one previously placed until they have no card with a higher score in hand.

Each card will have a power level assigned to it before the game starts. The individual power levels of the cards are stored in a triplestore and are calculated based on the metadata received from various established external APIs. The quiz given will also be based on the cards’ metadata and will reward the user with better cards, provided he answers correctly to the questions.

2. Start Menu

Once the player enters the website, he will be greeted with the main menu, which has the following section:

  • 1: button that starts the game
  • 2: used for selecting the number of rounds the user will play
  • 3: how many cards the user will have in hand each round
  • 4: button that redirects to the High Scores page
  • 5: button that redirects to the Wiki page

2.1. User High Score Menu

Clicking the “View scores” button from the previous page will redirect the user to the User High Scores view. Here, the user can see all the scores other players obtained, including himself.

2.2. GaDa Card Wiki menu

The GADA Card Wiki menu can be accessed by clicking the View Wiki button in the main menu. The page contains a list of all the cards extracted through the 4 APIs alongside details about them, including the name, Universe, Score, and a button with a more detailed view of the cards.

By pressing the “Details” button on the right side of the screen, the user can get more information about that particular card.

3. Card game menu

The card game menu has the following sections:

  • 1: Exit game button, which returns the player to the main menu
  • 2: Current round display, so the player can keep track of the number of rounds remaining
  • 3: Play card button, which is available after a player has selected a card
  • 4: Skip turn button, which is only available if the chain has an odd length
  • 5:User cards column, which has all the player cards
    • 5.1: User score
    • 5.2 Example of one of the cards, containing various information, such as universe,avatar,score and name
  • 6: Middle column:where the chain of cards is placed
    • 6.1: Total chain score: containing how many points the user/computer gains if he wins the chain
    • 6.2:Selected card: displays what the currently selected card by the player is, as well as the dynamic score after the type bonuses are applied
    • 6.3 & 6.4: Current Chain Top: the card that is currently at the top of the pile, with the score after type bonuses are applied, as well as name,universe and avatar
  • 7: Computer column: where the cards of the computer are displayed
    • 7.1: Computer’s score


After a short delay while the game creates a deck, the card_game view will be presented to the player. Here, the user will take turns with the computer to play the card game. The rules are simple:

  • You can only add cards with a higher score than the card that is currently on top of the chain.
  • If you place a card with a lower score, the chain ends and the opponent wins the chain score.
  • A chain cannot end unless its length is an even number.
  • Some cards will have higher dynamic score depending on their types.
  • The winner of the chain receives points equal to the number shown at the total chain score section

3.1. Selecting a card

The player and the computer will then take turns in placing cards on top of the chain until the chain itself ends. Players select a card first by clicking on it, which will make its border turn green. After the border turns green, the selected card section in the middle(6.2) will be updated based on its type and, if the card has a higher score than the Current Chain top(6.3), the play button(4) will become available. Pressing it will remove the card from your hand and put it on top of the chain. The computer will then make its move and place a card.

3.2. Winning/Losing a chain

Based on how the user performs, whenever a chain ends, the player will receive a message on top of the screen letting him know wherever he has won or lost the chain and the number of points he/the computer has won.

4. Quiz menu

After finishing each round (except the last one), the user will receive an alert that lets him know that the round has ended, and then he will be redirected to the quiz game where questions will appear about the characters from the cards he played in the previous round. He will, then, have to choose the correct answer(s) based on what he remembered from the last round by pressing the “Check” button. Depending on the number of cards he has chosen for each round, the user may have to answer one or more questions.

4.1. Question answer colouring

After pressing the “Check” button, the game will verify whether the answer to the question is correct or not. If the user was right, then the answers he has chosen will turn green and if was wrong, the answer will turn red.

4.2. Multiple choice questions

Sometimes, the user will have a multiple-answer question, where he will only win if he chooses all the correct ones. If he chooses at least one wrong answer or misses choosing a correct one, he will lose the quiz. Besides getting better cards if he wins the quiz, the user will also gain a small number of preset points and, conversely, lose points if he answers the quiz question wrong.

4.3. End of Quiz alert

Regardless of the outcome, after answering all the questions, an alert will be displayed on top of the screen letting the user know whether he has won or lost the quiz, as well as how many points he has won or lost.

5. End Menu

After all the cards from the deck have run out, the player will receive an alert letting him know that the game has ended. Afterward, he will be redirected to the End Menu which has the following elements:

  • 1: Submit score section, where the user can enter his Username and submit his score to the leaderboard.
    • 1.1: Field where the user can insert his username.
    • 1.2: Submit button for adding his score to the leaderboard
    • 1.3: Exit button for returning to the main menu
  • 2: Final panel, which contains:
    • 2.1: User’s final score
    • 2.2: Game status(win/lose/draw)
    • 2.3: Computer’s final score

If the user chooses to insert his name into the field and press the submit button, his score will be saved and will be shown the next time he visits the Leaderboard in the main menu.

5.1. Name Exists Alert

If the user chooses to submit a name that already exists on the leaderboard, he will receive an alert letting him know that he should enter a different name.
